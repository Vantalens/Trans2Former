## 测试编写指南

### 分支覆盖测试模式

```javascript
// 1. if/else 两个分支
test("函数名 - 成功路径", () => {
  // 测试 if 条件为真
});

test("函数名 - 失败路径", () => {
  // 测试 if 条件为假（else 分支）
});

// 2. 三元运算符
test("函数名 - 条件为真", () => {
  // condition ? A : B - 测试返回 A
});

test("函数名 - 条件为假", () => {
  // condition ? A : B - 测试返回 B
});

// 3. 逻辑运算符短路
test("函数名 - 第一个条件满足", () => {
  // A || B - 测试 A 为真（不执行 B）
});

test("函数名 - 第二个条件满足", () => {
  // A || B - 测试 A 为假，B 为真
});

test("函数名 - 两个条件都满足", () => {
  // A && B - 测试 A 和 B 都为真
});

test("函数名 - 第一个条件不满足", () => {
  // A && B - 测试 A 为假（短路，不执行 B）
});

// 4. switch 语句
test("函数名 - case A", () => {
  // 测试 case 'A'
});

test("函数名 - case B", () => {
  // 测试 case 'B'
});

test("函数名 - default", () => {
  // 测试 default 分支
});

// 5. try/catch
test("函数名 - 正常执行", () => {
  // 测试 try 块成功
});

test("函数名 - 异常处理", () => {
  // 测试 catch 块
});
```
