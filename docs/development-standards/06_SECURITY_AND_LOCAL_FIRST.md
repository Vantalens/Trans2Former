# Security And Local First

版本：v0.2.0
状态：生效  
最后更新：2026-04-30

## 底线

Trans2Former 的文档处理只能在用户设备上执行。项目不提供云端文档处理、远程转换、远程 OCR、远程转写或远程 AI 增强。默认不上传、不遥测、不留存用户文档内容。

## 默认禁止

- 默认上传文档、图片、音频、转换结果或错误详情。
- 上传文件名、文档片段、错误日志或转换结果。
- 接入远程转换 API、云端 OCR API、云端转写 API、云端 AI API、分析 SDK 或遥测 SDK。
- 默认把用户文档正文写入 localStorage、IndexedDB 或日志。
- 默认加载会向第三方发送用户内容的插件。
- 把 URL / YouTube extraction 作为格式路线。

## 插件安全要求

- 插件 manifest 必须声明 `localOnly`、`remoteCapable`、`networkRequired`。
- 插件安装模式可以联网下载代码和资源，但不得接触用户文档。
- 文档处理模式可以接触用户文档，但必须禁联网。
- local-only 插件不得调用网络 API。
- remote-capable 插件不得用于文档处理主线。
- 插件下载只下载代码和资源，不上传用户文件、文件名、片段或错误详情。
- 插件缓存必须可清理。
- 本地模型只能作为手动安装、手动启用、可删除插件，不能进入核心包，不能上传数据。
