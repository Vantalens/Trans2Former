# Security And Local First

版本：v0.3.0
状态：生效  
最后更新：2026-06-15

## 底线

Trans2Former 的文档处理只能在用户设备上执行。项目不提供云端文档处理、远程转换、远程 OCR、远程转写或远程 AI 增强。默认不上传、不遥测、不留存用户文档内容。

## 默认禁止

- 默认上传文档、图片、音频、转换结果或错误详情。
- 上传文件名、文档片段、错误日志或转换结果。
- 接入远程转换 API、云端 OCR API、云端转写 API、云端 AI API、分析 SDK 或遥测 SDK。
- 默认把用户文档正文写入 localStorage、IndexedDB 或日志。
- 默认加载会向第三方发送用户内容的模块或资源。
- 把 URL / YouTube extraction 作为格式路线。

## 模型缓存与本地模块安全要求

- OCR、layout、table、Repair Engine 和 verification 代码可以核心内置，但默认运行不得联网。
- 大模型和重资源不得进入默认安装包；首次启用时按需下载到本地 `model-cache`，并校验 SHA-256。
- model-cache 下载只获取模型资源，不上传用户文件、文件名、片段或错误详情。
- 文档处理、预览、编辑、转换和检验阶段必须禁联网。
- model-cache 条目必须可查看、可清理、可重新导入。
- 任何 remote-capable 能力不得进入文档处理主线。
- 本地模型启用后只能在用户设备上运行，不能上传数据。
