# Quality Gates

版本：v0.2.0
状态：生效  
最后更新：2026-06-15

## 默认验证

每轮开发默认运行：

```bash
npm test
```

当前测试链必须覆盖：

- 核心转换 smoke、快照、能力矩阵和格式完整性。
- Worker payload、浏览器入口、工作台队列、UI 可访问性和桌面壳边界。
- 本地安全、model-cache、OCR / PP-OCRv5、Repair Engine 和三层检验。
- 资源预算、响应性、发布产品化和 release readiness。

## 开发完成门禁

- 当前任务验收通过。
- `DEVELOPMENT_TASKS.md` 已更新。
- 文档入口和专题文档没有冲突。
- 没有新增未批准依赖。
- 没有默认上传、遥测、远程转换或用户内容持久化。

## 核心内置模块门禁

新格式、OCR/layout/table、Repair Engine 或验证能力进入主流程前必须满足：

- 能力、体积、依赖、安全模式和加载方式有明确文档或 manifest。
- 代码可以核心内置；大模型和重资源必须走 model-cache 按需获取，不进入默认安装包。
- 加载失败有用户可读错误和降级路径。
- 用户内容不会因为下载模型资源或本地模块而被上传。
- `resource-budget-test` 能阻止重依赖进入核心路径。
- 至少有样例、快照或 smoke 覆盖。

## 基础包门禁

热门基础格式可以免下载内置，但必须满足：

- 属于高频转换路径。
- 不引入默认重依赖。
- 不需要默认网络访问。
- 不突破核心资源预算。
- 有样例、快照、warnings 和安全覆盖。

## 发布门禁

- P0 问题为 0。
- 核心流程可手工完成：上传、预览、转换、取消、下载。
- 安全策略和资源预算测试通过。
- CHANGELOG 已记录用户可见或开发者可见变化。
