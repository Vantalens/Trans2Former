# PDF 预览空白问题说明

## 问题描述
转换生成 PDF 后，在浏览器中预览显示空白页面。

## 根本原因

当前 PDF 生成器使用 **系统字体引用** 而不是嵌入字体：

```
/BaseFont /STSong-Light
/Encoding /UniGB-UTF16-H
```

这意味着：
- PDF 文件本身是有效的
- 但依赖查看器系统中的 `STSong-Light` 字体
- 如果系统没有安装对应的中文字体，PDF 预览会显示空白

## 验证方法

1. **下载 PDF 文件**（点击"下载二进制输出"按钮）
2. **用其他 PDF 阅读器打开**：
   - Adobe Acrobat Reader
   - Foxit Reader
   - 或其他桌面 PDF 阅读器

这些专业 PDF 阅读器通常内置了中文字体支持，可以正常显示内容。

## 解决方案

### 短期方案
1. 下载 PDF 后用桌面 PDF 阅读器查看
2. 或者转换为其他格式（HTML、DOCX 等）进行预览

### 长期方案（需要开发）
需要修改 PDF 生成器以**嵌入字体子集**：

1. **嵌入 TrueType/OpenType 字体**
   - 需要引入字体文件（例如 Noto Sans CJK）
   - 使用 `/FontFile2` 或 `/FontFile3` 嵌入字体数据
   - 只嵌入文档中实际使用的字符（字体子集）

2. **技术挑战**：
   - 字体文件体积大（完整字体几 MB）
   - 需要字体子集化算法
   - 需要生成正确的 `/CIDToGIDMap`
   - 浏览器环境中字体处理复杂

3. **参考实现**：
   - jsPDF + 字体插件
   - PDFKit
   - pdfmake

## 临时测试方法

如果需要验证 PDF 内容是否正确生成：

```bash
# 使用 Node.js 检查 PDF 内容
node check-pdf-content.js
```

输出应显示：
- ✓ 是 PDF
- ✓ 包含页面
- ✓ 包含字体
- stream 数量 > 0

## 相关 Issue

这是一个已知限制，不是 Bug。PDF 生成功能正常工作，只是预览依赖系统字体。
