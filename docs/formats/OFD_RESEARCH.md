# OFD Research

版本：v0.3.0
状态：战略攻坚路线
最后更新：2026-05-01

## 定位

OFD 是 Trans2Former Desktop 的战略攻坚格式，面向中国政务、公文、电子票据和长期归档场景。它不是用来凑格式数量的边缘项，而是用来拉开与大多数通用转换器差距的高价值方向。

OFD 的目标路线是：

```text
本地解析
-> DocumentModel
-> QualityReport / Warnings
-> 本地 PNG/PDF 高保真渲染
-> 后续结构化编辑与导出
```

OFD 已进入核心 L0 本地能力路线。高保真渲染和 OCR/layout/table 相关能力继续通过核心本地按需加载模块推进，处理阶段必须 `local-only-no-network`，不提供插件安装入口。

## 为什么必须攻坚

- 市面上绝大多数通用转换器对 OFD 支持不足，尤其缺少本地、高质量、可解释的处理能力。
- OFD 关联政务、公文、电子票据和归档场景，用户对隐私、本地处理和保真度要求更高。
- OFD 高保真涉及字体、字形、矢量、签章、分页和版面一致性，正是专业转换器应当攻克的领域。
- 一旦建立公开样例库、质量报告和可解释降级机制，OFD 会成为 Trans2Former 的差异化壁垒。

## 攻坚原则

- 不以“能打开”作为验收；必须有公开样例、对照输出、质量指标和 warnings。
- 不走云端捷径；不得接入第三方 OFD 转换 API、云端 OCR、云端签章验证或远程日志上传。
- 不绕过 DocumentModel；OFD 文本、页面、图片、附件、字体和签章信息必须进入统一审计结构。
- 不把重渲染器塞进首屏启动路径；高保真能力通过可禁用、可回滚的核心本地模块提供。
- 不保存真实用户文件作为样例；样例必须公开、无敏感信息、有授权和可复现对照。

## 分阶段路线

## 能力等级

| 等级 | 目标 | 验收 |
| --- | --- | --- |
| OFD-L0 | 容器、manifest、metadata、页面数量、资源索引 | 公开样例可读取，输出 QualityReport |
| OFD-L1 | 页面树、文本对象、图片对象、附件列表到 DocumentModel | 结构快照稳定，warnings 可解释 |
| OFD-L2 | 本地 PNG/PDF 渲染基础 | 文本、图片、页面尺寸、多页输出有视觉对照 |
| OFD-L3 | 字体、矢量、签章、页面定位高保真专项 | 公开样例视觉差异可量化，降级项可定位到 block/page |
| OFD-L4 | 回归样例库、视觉 diff、chunked 等价和资源预算 | 核心模块可禁用、清理、回滚，处理阶段禁联网 |

## 高保真验收口径

- 必须提供公开样例和对照输出。
- 必须区分结构保真、视觉保真、资源保真和安全状态。
- 必须记录字体缺失、签章未验证、矢量近似、页面定位降级等原因。
- 必须支持质量报告导出，便于用户知道哪些页面或块发生降级。
- 不允许把不可验证的“高保真”写成已完成能力。

### OFD-L0：样例和容器基线

- 建立 `samples/ofd/README.md` 样例登记表。
- 收集公开、无敏感信息、有授权、有对照输出的 OFD 样例。
- 读取容器、manifest、metadata、页面数量、资源索引和附件清单。
- 输出基础 QualityReport，不渲染也要解释缺失能力。

### OFD-L1：DocumentModel 提取

- 提取页面树、文本对象、图片对象、附件列表和基础 metadata。
- 建立 `DocumentModel` 映射：页面分隔、段落、图片资源、附件引用、source span。
- 输出 block-level warnings：字体缺失、签章未验证、矢量未渲染、页面定位降级。

### OFD-L2：本地 PNG/PDF 渲染

- 建立本地 OFD -> PNG/PDF 渲染实验。
- 支持基础文本、图片、页面尺寸、坐标定位和多页输出。
- 建立视觉快照和对照输出，避免只做文本提取。
- 建立资源预算：本地模块体积、字体缓存、运行内存和渲染耗时。

### OFD-L3：高保真专项

- 字体和字形：缺失字体替代、嵌入字体读取、字形定位。
- 矢量图形：路径、线条、填充、透明度和裁剪。
- 签章：展示、完整性说明、未验证状态 warning。
- 页面版式：分页、坐标、层叠顺序、图片透明度和资源引用。

### OFD-L4：质量回归和核心模块化

- 建立公开回归样例库和视觉 diff。
- 建立 direct vs chunked 大文件等价测试。
- 建立 OFD 核心模块权限、完整性校验和 processing no-network 测试。
- 在 GUI 中展示 OFD 支持等级、质量报告、降级项和资源预算。

## 安全要求

- 不上传 OFD 文件、文件名、文档片段、签章信息、错误日志或转换结果。
- 不提供插件安装模式。
- 文档处理、编辑、预览和导出阶段必须禁联网。
- OFD 重能力模块必须可删除、可禁用、可回滚，并声明资源预算。
- 本地字体缓存、渲染缓存和版本历史必须用户可控。

## 推荐 warning codes

- `OFD_FONT_MISSING`
- `OFD_SIGNATURE_NOT_VERIFIED`
- `OFD_VECTOR_APPROXIMATED`
- `OFD_PAGE_LAYOUT_APPROXIMATED`
- `OFD_ATTACHMENT_SKIPPED`
- `OFD_RENDERER_RESOURCE_LIMIT`
- `OFD_CHUNKED_MERGE_DIFFERENCE`
