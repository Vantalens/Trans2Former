# OFD Research

版本：v0.1.0
状态：远期研究
最后更新：2026-04-30

## 定位

OFD 是 Trans2Former 的远期专业格式研究方向，归类为 P4+ / 政务格式 / 高保真渲染研究。它不进入近期核心包，也不作为近期 `format-plugin` MVP。

## 为什么保留

- 市面上绝大多数通用转换器对 OFD 支持不足。
- OFD 符合中国政务、公文和电子票据等专业场景。
- 如果能实现本地、高质量、可解释的 OFD 处理，会形成明显差异化。

## 为什么不进近期实现

- OFD -> PDF / 图片高保真涉及字体、字形、版面、分页、矢量图形、签章和渲染一致性。
- 浏览器端完整渲染器和解析器资源占用较大。
- 缺少足够公开样例、对照输出和质量评估基准时，容易做成低质量“能跑”功能。
- 当前 P0-P3 的主战场仍是 DOCX、XLSX、EPUB、PDF 文本提取和 PPTX。

## 正确路线

1. 收集公开、可测试、无敏感信息的 OFD 样例。
2. 建立 OFD capability note：支持范围、禁止承诺、降级说明。
3. 研究本地解析路线：容器结构、页面树、字体、图片、矢量、签章和附件。
4. 建立 OFD -> DocumentModel 的可解释提取实验，优先提取文本、页面、图片和 metadata。
5. 建立 OFD -> PNG/PDF 高保真渲染风险评估，不先承诺 Office 反向转换。
6. 若进入实现，必须作为 `format-plugin` 或本地渲染器插件，满足 no-network processing policy。

## 安全要求

- 不上传 OFD 文件、文件名、文档片段、签章信息或错误日志。
- 插件安装可以联网下载插件代码，但不能接触用户 OFD 文档。
- 文档处理阶段必须禁联网。
- OFD 插件必须可删除、可禁用，并声明资源预算。
