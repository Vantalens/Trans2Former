# Trans2Former

> Многоформатный инструмент конвертации документов с приоритетом локальной обработки

[简体中文](README.md) | [English](README.en.md) | [日本語](README.ja.md) | [한국어](README.ko.md) | [Français](README.fr.md) | [Español](README.es.md) | [Deutsch](README.de.md) | [Português](README.pt-BR.md) | [Русский](README.ru.md) | [العربية](README.ar.md)

Trans2Former — настольный инструмент конвертации документов: 14 входных форматов, 11 выходных форматов, все преобразования выполняются локально на вашем компьютере. Без загрузки файлов на сервер, без зависимостей от Office / LibreOffice / Pandoc, с генерацией интерпретируемого отчёта о качестве для каждой конвертации.

## Особенности

- **Локальность прежде всего**: конвертация, OCR и рендеринг формул выполняются полностью офлайн; на этапе обработки документов сетевой доступ запрещён
- **Проверяемая конвертация**: трёхуровневая верификация — diff по правилам, визуальное сравнение SSIM, обратное OCR-считывание; результаты записываются в единый отчёт о качестве и отображаются в рабочей области
- **Локальный OCR**: встроенный PP-OCRv5 (ONNX Runtime, WebGPU / WASM) с коррекцией ориентации, исправлением наклона, адаптивным шумоподавлением, распознаванием структуры макета и оценкой качества
- **Рендеринг формул**: локальная вёрстка KaTeX для `$...$` / `$$...$$`
- **Высокая производительность**: параллельный конвейер на Web Worker, без искусственных ограничений на размер файлов
- **Ноль зависимостей во время выполнения**: ядро конвертации не требует никакого внешнего офисного ПО

## Поддерживаемые форматы

| Категория | Вход (14 форматов) | Выход (11 форматов) |
| --- | --- | --- |
| Документы | Markdown, HTML, TXT, DOCX, PDF, EPUB | Markdown, HTML, TXT, DOCX, PDF, EPUB |
| Данные | JSON, CSV, XML, XLSX | JSON, CSV, XML, XLSX |
| Презентации | PPTX | PPTX |
| Изображения | PNG (OCR-распознавание) | — |
| Экспериментальные | DOC (только извлечение текста), OFD (ранняя предварительная версия) | — |

Популярные маршруты: Markdown ↔ HTML · DOCX → Markdown · PDF → Markdown · XLSX ↔ CSV · HTML → PDF

Полная матрица конвертации: [docs/CONVERSION_PATHS.md](docs/CONVERSION_PATHS.md).

## Быстрый старт

```bash
npm install   # установка зависимостей
npm start     # запуск, откройте в браузере http://localhost:3000
npm test      # запуск набора тестов (28 скриптов)
```

Настольное приложение (Tauri 2) и сборка релиза:

```bash
npm run desktop:dev       # режим разработки настольного приложения
npm run release:prepare   # сборка релизного пакета
```

## Основные локальные возможности

Расширенные возможности встроены непосредственно в ядро без механизма плагинов; ресурсы моделей не хранятся в git — vendor-скрипты загружают их из закреплённых источников с проверкой SHA-256 ([scripts/paddleocr-models.manifest.json](scripts/paddleocr-models.manifest.json)), они распространяются вместе с релизным пакетом и работают «из коробки».

- **Локальный OCR PP-OCRv5**: детекция + распознавание + классификация ориентации для изображений и сканированных PDF, включая исправление наклона, шумоподавление, объединение макета и оценку уверенности
- **Лёгкий OCR Tesseract.js**: опциональный движок, активируется импортом tessdata в центре безопасности
- **Трёхуровневая верификация конвертации**: diff по правилам + SSIM + обратное OCR-считывание; результаты интерпретируемы и допускают плавную деградацию
- **Математический рендеринг KaTeX**: без сетевого доступа

Подготовка среды выполнения OCR:

```bash
npm install onnxruntime-web
npm run vendor:onnx
npm run vendor:paddle
```

Протокол верификации в браузере: [docs/PP_OCRV5_BROWSER_VERIFICATION.md](docs/PP_OCRV5_BROWSER_VERIFICATION.md).

## Безопасность данных

- Файлы, имена файлов, результаты конвертации и журналы ошибок никуда не передаются
- На этапе обработки документов сетевой доступ запрещён
- Не используются сторонние API конвертации и аналитические SDK

Полная политика: [docs/SECURITY_POLICY.md](docs/SECURITY_POLICY.md).

## Известные ограничения

1. Некоторые сложные стили не сохраняются полностью при межформатной конвертации
2. Анимация и диаграммы PPTX пока не поддерживаются
3. OCR ограниченно распознаёт выраженный курсив и декоративные шрифты; входные форматы DOC / OFD являются экспериментальными
4. Сверхбольшие архивы ZIP64 пока не поддерживаются
